# pi-daily 顶层设计

## 产品定位

pi-daily 是一个 Pi Coding Agent 扩展，用本地 Pi session 活动自动生成每日工作日报。

它回答一个问题：

> 今天我通过 Pi 实际推进了哪些事情？

## 目标用户

- 高频使用 Pi 处理研发、产品、文档和自动化工作的个人用户。
- 需要把零散 agent session 整理成日报、周报或复盘素材的团队成员。
- 希望保留本地隐私，不把完整 session 原文上传到第三方服务的用户。

## 核心原则

1. **本地优先**：默认只读本地 session 文件，不上传原文。
2. **证据优先**：日报来自真实 session 条目，而不是凭空总结。
3. **AI 在结构化事实之后**：先本地提取结构化事实，再把脱敏后的摘要交给 AI，总结失败时回退本地规则版。
4. **可控输出**：默认直接输出到对话区；只有用户显式请求时保存文件。
5. **隐私安全**：默认避免输出长原文、密钥、token、私钥等敏感片段。

## 不做什么

MVP 不做：

- 自动发送日报到飞书、邮件、Slack。
- 后台定时任务。
- 云端同步 session。
- 对完整 session 原文做外部 LLM 分析。
- 修改或删除 Pi 原始 session 文件。

## 日报输出目标

日报应包含：

- 日期和统计概览。
- 活跃项目 / 工作目录。
- 今日主要任务。
- 关键产出。
- 读写文件和工具活动概览。
- 错误、阻塞和待跟进事项。
- 可复制的 Markdown 内容。
