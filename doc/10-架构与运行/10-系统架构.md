# pi-daily 系统架构

## 当前形态

pi-daily 是单包 Pi extension，入口为 `index.ts`。

```text
pi-daily/
├── index.ts          # 注册 /daily 命令
├── src/
│   ├── args.ts            # 解析命令参数
│   ├── ai-summary.ts      # AI 总结与模型回退；按事项输出日报
│   ├── daily.ts           # 生成、渲染和保存日报
│   ├── session-scan.ts    # 扫描 session JSONL 文件
│   ├── session-extract.ts # 解析消息、工具调用、文件活动
│   ├── report-model.ts    # 聚合日报模型
│   ├── markdown-render.ts # 渲染 Markdown 日报
│   └── redact.ts          # 敏感信息脱敏
├── tests/            # Node.js 测试
└── doc/              # 产品、架构、路线图
```

## 数据来源

Pi session 默认位于：

```text
~/.pi/agent/sessions/
```

session 文件为 JSONL 格式：

- `type=session`：session 元数据，包含 `cwd`。
- `type=message`：用户、assistant、toolResult 等消息。
- `type=custom` / `custom_message`：扩展状态或上下文。
- `type=compaction` / `branch_summary`：压缩和分支摘要。

当前实现已经具备：

- `/daily` 命令注册与参数解析。
- session JSONL 扫描与坏行容错。
- 活动提取、项目聚合、Markdown 渲染。
- `完成事项` 优先来自 assistant 的完成结论，不回放原始 user prompt。
- `待跟进` 来自未闭环请求、进度结论和阻塞项。
- 默认优先用 AI 基于脱敏后的结构化事实生成日报，失败时回退本地规则版。
- AI 日报按项目分组，每个项目单独输出 `进展`、`产出`、`阻塞 / 风险`、`待跟进`。
- 运行逻辑使用直接 TypeScript 模块，源码和测试均保持 `.ts`。
- 默认将日报作为一条自定义消息直接注入对话区，避免 widget 截断。
- `--save` 可写入日报文件。
- 基础测试覆盖主链路。

MVP 采用手动 JSONL 解析，而不是依赖内部 `SessionManager`。原因：

- 可以跨项目扫描全部 session。
- 对 Pi 内部 API 变化更稳。
- 更容易做坏行容错和离线测试。

## 命令设计

MVP 注册：

```text
/daily
/daily 2026-06-12
/daily --today
/daily --save
/daily --project current
```

命令行为：

1. 解析参数，确定日期和扫描范围。
2. 扫描 session 文件。
3. 筛选目标日期有活动的条目。
4. 聚合为日报模型。
5. 渲染 Markdown。
6. 将日报作为自定义消息直接发送到对话区；如传 `--save`，保存到配置的输出目录。
7. AI 总结失败时，在日报底部输出 fallback 诊断信息。
8. AI 总结直接使用当前会话模型；不额外读取模型配置文件。

## 输出位置

MVP 建议默认保存目录：

```text
~/Documents/pi-daily-reports/YYYY-MM-DD.md
```

保存动作必须由用户显式触发。

## 隐私与安全

- 不修改原始 session。
- 不默认输出完整 tool result。
- 对常见 secret pattern 做脱敏。
- 长文本截断，保留任务标题和证据摘要。

## 验证

```bash
npm run check
npm test
```

测试应覆盖：

- JSONL 坏行容错。
- 当天活动过滤。
- 用户消息、assistant tool call、toolResult 提取。
- Markdown 输出结构。
- 敏感信息脱敏。
