# pi-daily 路线图

## 命名

注册名采用：`pi-daily`。

理由：

- npm 当前未发现 `pi-daily` 已被注册。
- 比 `pi-daily-report` 更短，适合 slash command 和品牌记忆。
- 后续可扩展到日报、周报、工作日志，而不局限于单一 report 输出。

## MVP：本地 session 每日日报

目标：用户运行 `/daily` 后，基于当天本地 Pi session 活动生成可复制 Markdown 日报。

### MVP 功能清单

- [x] 注册 `/daily` 命令。
- [x] 支持默认生成今天日报。
- [x] 支持指定日期：`/daily YYYY-MM-DD`。
- [x] 支持跨凌晨统计窗口：`--day-start HH:mm`、`--since/--until HH:mm`、`--from/--to YYYY-MM-DDTHH:mm`。
- [x] 扫描 `~/.pi/agent/sessions/**/*.jsonl`。
- [x] 过滤目标日期有活动的 session。
- [x] 按 `cwd` 和 session 聚合活动。
- [x] 提取用户任务、assistant 关键文本、tool call、tool result 摘要。
- [x] 识别文件活动：`read`、`write`、`edit`、`bash` 中的文件路径。
- [x] 统计错误和阻塞：失败 tool result、非 0 bash exit code、包含 blocker/error 的片段。
- [x] 渲染 Markdown 日报。
- [x] 默认优先用 AI 基于脱敏后的结构化事实生成日报，失败时回退本地规则版。
- [x] 直接将日报发送到对话区。
- [x] 支持 `--save` 保存到 `~/Documents/pi-daily-reports/`。
- [x] 添加基础测试和 README。

> 注：当前版本已完成 MVP 主链路，可直接从本地 session 生成日报。
>
> 版本级实施详情见：`../40-版本实施方案/40-V0.0.1-本地session日报实施方案.md`。

### MVP 输出结构

```md
# YYYY-MM-DD 工作日报

## 今日概览

## 项目/事项一：<项目名或事项名>
### 进展
### 产出
### 阻塞 / 风险
### 待跟进

## 项目/事项二：<项目名或事项名>
### 进展
### 产出
### 阻塞 / 风险
### 待跟进

## 跨项目观察

## 工具与文件活动
```

### MVP 验收标准

- 在无真实 session 的临时目录测试可通过。
- 在真实 `~/.pi/agent/sessions` 上运行不崩溃。
- 不输出明显密钥原文。
- 默认不写文件。
- `npm run check` 和 `npm test` 通过。

## V0.2：更好用的命令参数

- [ ] `/daily --project current` 只看当前工作目录相关 session。
- [x] `/daily --since HH:mm` 支持从当天某个时间开始，可配合 `--until` 跨凌晨。
- [ ] `/daily --range YYYY-MM-DD..YYYY-MM-DD` 支持多日区间。
- [ ] `/daily --format compact|full` 支持简版 / 详细版。
- [ ] `/daily --copy` 复制到剪贴板。

## V0.3：AI 润色增强

- [x] 默认使用当前 Pi 对话模型总结日报。
- [x] 只发送脱敏后的结构化摘要，不发送完整 session 原文。
- [x] 按真实项目和默认工作区中的具体事项分组输出。
- [x] 支持更稳定的输出语言 / 风格控制（11 种语言本地化，参考 pi-compaction-i18n 模式）。
- [ ] 支持摘要质量评分与自动重试。
