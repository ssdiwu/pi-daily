# V0.0.1 本地 session 日报实施方案

## 1. 结论

pi-daily 的第一个可交付版本定义为 **V0.0.1**：从本地 Pi session 活动生成每日 Markdown 日报，并通过 `/daily` 命令在 Pi 内展示，支持显式保存。

> 当前代码已经实现该版本主链路，本方案作为权威实施记录。

## 2. 目标

### 2.1 用户目标

用户运行：

```text
/daily
```

即可看到当天由本地 session 汇总出来的日报，知道自己今天通过 Pi 实际推进了什么。

### 2.2 产品目标

- 先把“从 session 生成日报”这件事做稳定。
- 不依赖外部云服务。
- 不要求 AI 再加工完整原文。
- 输出足够可读、可复制、可保存。

## 3. 范围

### 3.1 本版本要做

- 注册 `/daily` 命令。
- 支持默认生成今天日报。
- 支持 `/daily YYYY-MM-DD`。
- 扫描 `~/.pi/agent/sessions/**/*.jsonl`。
- 过滤目标日期有活动的 session。
- 按 `cwd` 和 session 聚合活动。
- 提取：
  - 用户任务
  - assistant 关键文本
  - tool call
  - tool result 摘要
  - 文件路径
  - 错误 / 阻塞
- `完成事项` 由 assistant 完成结论汇总，不直接回放 user 原话。
- `待跟进` 由未闭环请求、assistant 进度结论和阻塞项汇总。
- 渲染 Markdown 日报。
- 默认优先用当前对话模型基于脱敏后的结构化事实生成日报，按真实项目和具体事项分组；失败时回退本地规则版，并在日报底部显示 fallback 诊断。
- 默认作为自定义消息直接发送到对话区。
- 支持 `--save` 保存到本地目录。
- 用测试覆盖主链路。

### 3.2 本版本不做

- 不做定时任务。
- 不做飞书 / 邮件 / Slack 自动发送。
- 不做云同步。
- 不做外部 LLM 重写完整 session。
- 不修改原始 session 文件。
- 不做周报 / 月报。
- 不做复杂标签体系和知识图谱。

## 4. 数据契约

### 4.1 输入

输入源为 Pi 本地 session JSONL 文件。

可接受的关键结构：

- `type=session`：包含 `cwd` 和 session 元数据。
- `type=message`：包含 `user` / `assistant` / `toolResult` / `bashExecution` 等活动。
- `type=custom`、`custom_message`、`compaction`、`branch_summary`：可忽略或只保留基础容错。

### 4.2 输出

输出为 Markdown，AI 总结默认结构：

```md
# YYYY-MM-DD 工作日报

## 今日概览

## 项目/事项一：<项目名或事项名>
### 进展
### 产出
### 阻塞 / 风险
### 待跟进

## 项目/事项二：<项目名或事项名>
### 进展
### 产出
### 阻塞 / 风险
### 待跟进

## 跨项目观察

## 工具与文件活动
```

### 4.3 保存路径

默认保存路径：

```text
~/Documents/pi-daily-reports/YYYY-MM-DD.md
```

保存动作必须由用户显式触发。

## 5. 实现拆分

### 5.1 命令层

`index.ts`

- 注册 `/daily`。
- 调用日报生成流程。
- 控制 UI 展示与保存。

### 5.2 参数层

`src/args.ts`

- 解析日期。
- 解析 `--save`。
- 解析 `--project current`。

### 5.3 扫描层

`src/session-scan.ts`

- 遍历 session 目录。
- 读取 JSONL。
- 容忍坏行。
- 返回文件、header、entries、errors。

### 5.4 抽取层

`src/session-extract.ts`

- 过滤目标日期。
- 提取 user / assistant / toolResult / bashExecution。
- 提取文件路径、错误、阻塞。

### 5.5 聚合层

`src/report-model.ts`

- 按 session、cwd、项目名聚合。
- 统计活跃项目、文件数、工具次数。

### 5.6 渲染层

`src/markdown-render.ts`

- 输出 Markdown。
- 控制 section 结构。
- 对敏感信息做最终脱敏。

### 5.7 脱敏层

`src/redact.ts`

- 处理常见 token、key、secret、password。
- 控制长文本长度。

## 7. 风险与约束

### 7.1 隐私风险

session 里可能包含敏感文本，因此必须默认脱敏，且不要默认导出完整原文。

### 7.2 解析风险

不同 Pi 版本可能写入不同 entry 结构，因此解析器必须容错，不能因为单条坏数据导致整个日报失败。

### 7.3 噪声风险

自动提取文件名、命令和结果片段可能引入噪声，因此日报应保留摘要而非全文。
