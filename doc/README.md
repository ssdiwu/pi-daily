# doc 文档导航

> 本目录保存 pi-daily 的产品原则、架构运行和路线图。做代码或产品决策前先读这里，避免实现和目标脱节。

## 阅读顺序

1. `00-产品与原则/00-顶层设计.md`
2. `10-架构与运行/10-系统架构.md`
3. `30-路线图/30-路线图.md`
4. `40-版本实施方案/README.md`

## 目录职责

| 目录 | 职责 | 是否权威 |
|---|---|---|
| `00-产品与原则/` | 产品定位、目标用户、隐私原则、日报边界 | 是，长期有效 |
| `10-架构与运行/` | 当前目录结构、数据流、命令、测试方式 | 是，当前实现权威 |
| `30-路线图/` | MVP 范围、后续版本计划、验收清单 | 是，路线图权威 |
| `40-版本实施方案/` | 版本级落地方案、输入输出契约、checklist | 是，执行权威 |

## 文档维护规则

- 改变命令、输出结构、配置、隐私策略或 session 解析口径时，同步更新相关文档。
- 路线图只描述阶段目标，不记录临时调试细节。
- 架构文档描述当前实现，不提前写未落地能力。
